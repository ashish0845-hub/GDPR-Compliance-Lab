# GDPR-Compliance-Lab
## GDPR Compliance Checklist

| GDPR Principle | Application Spec # | Status | Evidence / Notes |
|---|---|---|---|
| Lawfulness (Art. 6) | Spec 2 | ✅ | Users must register and log in to use the system. |
| Child Consent (Art. 8) | Spec 6 | ✅ | System enforces a minimum age of 15 for booking. |
| Data Minimization | Spec 8 | ✅ | Public view hides identity; only shows resource availability. |
| Right to Erasure | Spec 5 | ✅ | Admin has the specific function to delete reserver accounts. |
| Privacy by Design | Spec 10 | ✅ | Privacy is built into the identity-masking display logic. |
| Access Control | Spec 3, 4 | ✅ | Clear separation between Reserver and Admin permissions. |

**Auditor Conclusion:** The system follows Privacy by Design (PbD) by ensuring the highest privacy settings are applied by default (identity masking).
